---
title: "Configure the AI model, context, and system prompt"
description: "Configure the OpenRouter model, conversation context window, temperature, and system prompt that define signal-bot-orx's chat behavior and personality."
---

Chat configuration controls which AI model the bot uses, how many turns of conversation history it retains per chat, and how its personality is defined through the system prompt. All of these settings are read from environment variables at startup.

## OpenRouter settings

These variables configure the connection to OpenRouter and the chat completion parameters.

<ParamField path="OPENROUTER_CHAT_API_KEY" type="string" required>
  Your OpenRouter API key for chat completions. Startup fails if this variable is empty or unset.
</ParamField>

<ParamField path="OPENROUTER_MODEL" type="string" default="openai/gpt-4o-mini">
  Any model identifier supported by OpenRouter, such as `anthropic/claude-3.5-sonnet` or `google/gemini-2.0-flash-001`. The default is `openai/gpt-4o-mini`.
</ParamField>

<ParamField path="OPENROUTER_BASE_URL" type="string" default="https://openrouter.ai/api/v1">
  Base URL for the OpenRouter API. Change this only if you are proxying requests through a custom endpoint.
</ParamField>

<ParamField path="OPENROUTER_TIMEOUT_SECONDS" type="number" default="45">
  How long (in seconds) to wait for a chat completion response before timing out. The client makes up to 3 total attempts on transient errors (429, 500, 502, 503, 504) before returning a timeout message to the chat.
</ParamField>

<ParamField path="OPENROUTER_MAX_OUTPUT_TOKENS" type="number" default="300">
  Maximum number of tokens in each model reply. Keep this low for chat use; it directly affects response latency and per-message cost.
</ParamField>

<ParamField path="OPENROUTER_HTTP_REFERER" type="string">
  Optional. Sent as the `HTTP-Referer` request header to OpenRouter. Useful for tracking usage in the OpenRouter dashboard.
</ParamField>

<ParamField path="OPENROUTER_APP_TITLE" type="string" default="signal-bot-orx">
  Sent as the `X-Title` request header to OpenRouter. Identifies your application in OpenRouter usage reports.
</ParamField>

<AccordionGroup>
  <Accordion title="Image generation settings (/imagine)">
    These variables enable the `/imagine` command, which uses OpenRouter's `modalities: ["image"]` chat completions endpoint.

    <ParamField path="OPENROUTER_IMAGE_API_KEY" type="string">
      API key for image generation calls. If left empty, the `/imagine` command is disabled.
    </ParamField>

    <ParamField path="OPENROUTER_IMAGE_MODEL" type="string">
      Model identifier to use for image generation. Must support the `image` modality on OpenRouter. If left empty, `/imagine` is disabled.
    </ParamField>

    <ParamField path="OPENROUTER_IMAGE_TIMEOUT_SECONDS" type="number" default="90">
      Timeout in seconds for image generation requests. Image models take significantly longer than text models, so the default is higher than `OPENROUTER_TIMEOUT_SECONDS`.
    </ParamField>

    <Tip>
      You can use the same API key for `OPENROUTER_CHAT_API_KEY` and `OPENROUTER_IMAGE_API_KEY` if you are comfortable sharing rate limits and billing across chat and image calls.
    </Tip>
  </Accordion>
</AccordionGroup>

## Conversation context

The bot maintains a separate in-memory conversation history for each group or direct message thread. These settings control how large that window is and how long it stays active.

<ParamField path="BOT_CHAT_CONTEXT_TURNS" type="number" default="6">
  Number of back-and-forth turns to keep per conversation. Each turn is one user message plus one assistant reply. Older turns are dropped when the limit is reached. A higher value improves coherence in long threads but increases the token count sent to the model on every request.
</ParamField>

<ParamField path="BOT_CHAT_CONTEXT_TTL_SECONDS" type="number" default="1800">
  How long (in seconds) a conversation's context remains active without new messages before it is reset. After 1800 seconds (30 minutes) of inactivity the history is cleared and the next message starts a fresh conversation.
</ParamField>

<ParamField path="BOT_CHAT_TEMPERATURE" type="number" default="0.6">
  Sampling temperature passed to the model. Lower values (closer to 0) produce more deterministic, factual replies. Higher values (closer to 1) produce more creative or varied responses.
</ParamField>

## System prompt

The system prompt defines the bot's personality and baseline instructions. The bot resolves the active system prompt from these three sources in priority order:

1. **`BOT_CHAT_SYSTEM_PROMPT` env var** — when this variable is non-empty, it overrides everything else.
2. **`src/signal_bot_orx/chat_system_prompt.md`** — a git-ignored local file created automatically from the distribution default on first startup. Edit this file for a persistent local override without touching the tracked codebase.
3. **`src/signal_bot_orx/default_chat_system_prompt.md`** — the distribution default, tracked in version control. Used as the final fallback if neither source above is present.

<ParamField path="BOT_CHAT_SYSTEM_PROMPT" type="string">
  Optional. When set, this value is used directly as the system prompt, bypassing both prompt files. Useful for deployment environments where file-based overrides are not practical.
</ParamField>

<Note>
  On first run, the bot copies `default_chat_system_prompt.md` to `chat_system_prompt.md` if the local file does not exist. Edit `chat_system_prompt.md` to customise the bot's personality without modifying the distribution default.
</Note>

## Text output

<ParamField path="BOT_CHAT_FORCE_PLAIN_TEXT" type="boolean" default="true">
  When `true`, the bot strips markdown formatting from model replies before sending them. This removes code fences, headers, bold/italic markers, bullet characters, and inline links, producing clean plain text suitable for Signal, WhatsApp, and Telegram.
</ParamField>

<ParamField path="BOT_MAX_PROMPT_CHARS" type="number" default="700">
  Maximum number of characters accepted from a user's message before it is truncated. Prompts longer than this limit are cut off at the character boundary to prevent unexpectedly large context payloads.
</ParamField>

<ParamField path="BOT_MENTION_ALIASES" type="string" default="@signalbot,@bot">
  Comma-separated list of mention strings that trigger the bot in group chats. The bot responds when any alias appears in the message text, in addition to metadata-based mention detection. In direct messages the bot always replies without requiring a mention.
</ParamField>

```bash
# Example: extend aliases for a custom bot name
BOT_MENTION_ALIASES=@signalbot,@bot,@mybot
```
