signal-bot-orx is a self-hosted webhook bot that bridges Signal, WhatsApp, and Telegram to OpenRouter-powered language models. Drop it into your group chats and get AI-powered replies triggered by mentions, along with slash commands for web search, image generation, and weather lookups — all without sending your messages to a third-party service you don't control.

## What signal-bot-orx does

signal-bot-orx listens for incoming webhooks from your messaging platform and routes messages through OpenRouter to a language model of your choice. In group chats, the bot only replies when mentioned by name or alias. In direct messages, every message triggers a reply.

## Get started in three steps

<Steps>
  <Step title="Install signal-bot-orx">
    Clone the repo and install with `uv sync --dev`, or install from PyPI with `pip install signal-bot-orx`.
  </Step>
  <Step title="Configure your environment">
    Copy `.env.example` to `.env`, set your `OPENROUTER_CHAT_API_KEY`, and configure at least one transport (Signal is enabled by default).
  </Step>
  <Step title="Run the bot">
    Load your environment with `source .env` and start with `uv run signal-bot-orx`. Point your Signal, WhatsApp, or Telegram webhook to the bot's endpoint.
  </Step>
</Steps>

<Note>
  signal-bot-orx requires a running [signal-cli-rest-api](https://github.com/bbernhard/signal-cli-rest-api) instance and an [OpenRouter](https://openrouter.ai) API key. See the [quickstart](/quickstart) for full prerequisites.
</Note>
