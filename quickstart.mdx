---
title: "Quickstart: install, configure, and run signal-bot-orx"
description: "Install signal-bot-orx, configure your environment, and have a working bot connected to Signal and responding to group mentions in under ten minutes."
---

This page walks you through every step required to go from zero to a running signal-bot-orx instance connected to Signal. By the end you will have the bot service listening for webhooks, responding to mentions in your Signal group, and passing messages through OpenRouter to a language model of your choice.

<Steps>

<Step title="Prerequisites">

Before you start, make sure you have the following:

- **`uv`** installed — the package manager used to run and install signal-bot-orx. See the [uv installation guide](https://docs.astral.sh/uv/).
- **`signal-cli-rest-api`** running and registered with your bot phone number. See [Signal setup](/setup/signal) for full instructions.
- **OpenRouter API key** — required for AI chat. Get one at [openrouter.ai](https://openrouter.ai).
- **OpenWeatherMap API key** — optional, required only if you want `/weather` and `/forecast` commands.

</Step>

<Step title="Installation">

Install signal-bot-orx using whichever method suits your workflow.

<CodeGroup>

```bash From source
git clone https://github.com/htmlgxn/signal-bot-orx.git && cd signal-bot-orx && uv sync --dev
```

```bash PyPI (pip)
pip install signal-bot-orx
```

```bash PyPI (uv)
uv tool install signal-bot-orx
```

</CodeGroup>

</Step>

<Step title="Configure">

Copy the example environment file and set your required values:

```bash
cp .env.example .env
```

Open `.env` and set at minimum these required variables:

```bash
OPENROUTER_CHAT_API_KEY=sk-or-...
SIGNAL_API_BASE_URL=http://127.0.0.1:8080
SIGNAL_SENDER_NUMBER=+15550001111
SIGNAL_ALLOWED_GROUP_IDS=your-group-id
```

If you want to skip allowlist enforcement during initial testing, you can set `SIGNAL_DISABLE_AUTH=true` instead of `SIGNAL_ALLOWED_GROUP_IDS`. Remove it before using the bot in production.

</Step>

<Step title="Start the bot">

Load your environment variables and start the service:

```bash
set -a
source .env
set +a
uv run signal-bot-orx
```

The bot binds to `127.0.0.1:8001` by default. If you need it accessible on the network (for example in a container or LAN deployment), set `BOT_WEBHOOK_HOST=0.0.0.0` in your `.env`.

</Step>

<Step title="Health check">

Verify the service is running:

```bash
curl -s http://127.0.0.1:8001/healthz
```

You should receive:

```json
{"status": "ok"}
```

If the command times out or returns an error, check that the process started without missing environment variable errors in the terminal output.

</Step>

<Step title="Configure the webhook">

Tell `signal-cli-rest-api` where to deliver incoming messages. Set its callback URL to:

```
http://<host>:8001/webhook/signal
```

For a local setup this is typically `http://127.0.0.1:8001/webhook/signal`. For a LAN or container deployment, replace `<host>` with the IP or hostname where signal-bot-orx is reachable from `signal-cli-rest-api`.

See [Signal setup](/setup/signal) for step-by-step instructions on configuring the callback in `signal-cli-rest-api`.

</Step>

<Step title="Test it">

Send a message in your Signal group that mentions the bot:

```text
@bot summarize the key decisions from this thread
```

The bot responds when the sender number or group ID matches your allowlist. In a direct message, no mention is needed — every message triggers a reply.

Try a search command:

```text
/search latest OpenRouter announcements
```

Or ask for the weather:

```text
/weather London
```

</Step>

</Steps>

<Tip>
  The bot exposes four endpoints once running:

  - `GET /healthz` — liveness check
  - `POST /webhook/signal` — Signal webhook receiver (always active)
  - `POST /webhook/whatsapp` — WhatsApp bridge receiver (active when `WHATSAPP_ENABLED=true`)
  - `POST /webhook/telegram` — Telegram webhook receiver (active when `TELEGRAM_ENABLED=true`)
</Tip>

For a complete walkthrough of registering a Signal number and configuring `signal-cli-rest-api`, see [Signal setup](/setup/signal).
