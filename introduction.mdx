signal-bot-orx is a webhook-driven bot that bridges Signal, WhatsApp, and Telegram to language models via OpenRouter. It is designed for developers and privacy-conscious users who want AI-powered group chat assistance without routing messages through a third-party service they don't control. Deploy it alongside `signal-cli-rest-api` and your bot is live in minutes, responding to mentions in group chats and every message in direct conversations.

<Note>
  signal-bot-orx is in active development. Some features may be broken or behave unexpectedly between updates.
</Note>

## How it works

signal-bot-orx runs as a lightweight FastAPI service that listens for incoming webhook payloads from your messaging platform. The flow for every interaction is:

1. A user sends a message in a Signal group (or WhatsApp/Telegram if those transports are enabled).
2. `signal-cli-rest-api` delivers the message as a `POST` request to the bot's `/webhook/signal` endpoint.
3. The bot checks whether the sender or group is allowlisted, then extracts the message text.
4. In group chats, the bot only responds when it is mentioned by name or alias. In direct messages, every message triggers a reply.
5. The bot sends the conversation history and the new message to OpenRouter, which forwards the request to your configured language model.
6. The model's reply is sent back through `signal-cli-rest-api` to the original group or user.

Slash commands (`/search`, `/news`, `/wiki`, `/images`, `/videos`, `/imagine`, `/weather`, `/forecast`) are intercepted before the LLM call and routed to their respective backends — DuckDuckGo search, OpenRouter image generation, or OpenWeatherMap.

## Architecture overview

signal-bot-orx is composed of a small set of cooperating services:

- **signal-cli-rest-api** — registers your Signal number, sends and receives messages, and delivers incoming payloads to the bot's webhook endpoint. You run and manage this service separately.
- **The bot service** — a FastAPI application (`signal-bot-orx`) that receives webhooks, applies access control, manages per-group conversation history, and orchestrates calls to downstream services.
- **OpenRouter** — the API gateway that routes chat and image requests to your chosen language model. You supply an API key; the model is configurable.
- **Optional services** — OpenWeatherMap provides weather data when `WEATHER_API_KEY` is set. A WhatsApp bridge (whatsapp-web.js) and the Telegram Bot API are supported as additional transports when enabled.

The bot process binds to `127.0.0.1:8001` by default. Set `BOT_WEBHOOK_HOST=0.0.0.0` in your `.env` to expose it on the network for container or LAN deployments.
